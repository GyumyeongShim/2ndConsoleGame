# 프로젝트: C++ 전략 JRPG (Grid-based Console Game)

## 1. 프로젝트 개요
- **장르:** 전략적 이동이 포함된 턴제 JRPG (참조: Crypt of the NecroDancer, Fire Emblem)
- **환경:** C++ / Windows Console (Double Buffering Renderer)
- **핵심 루프:** Title -> Town -> Field/Dungeon (A* Monster Encounter) -> Battle -> Town (Cycle)

## 2. 현재 아키텍처 및 파일 구조
- **Engine:** `Actor`, `Component`(Stat, Skill, Inventory), `Render`(Canvas, Camera, ScreenBuffer), `UI` 시스템이 분리된 모듈형 구조.
- **Game:** `BattleLevel`, `TitleLevel` 등 레벨 단위 관리 및 `World/TileMap` 기반의 맵 시스템.
- **Data:** `.json` (Item, Monster, Player, Skill) 파일을 통한 데이터 주도(Data-Driven) 설계.
- **ThirdParty:** `json.hpp` 사용 중.

## 3. 구현 목표 (Level 구성)
기획에 따라 다음과 같이 4개의 주요 레벨로 구분하여 개발 진행:
1. **TitleLevel:** 게임 시작 및 설정.
2. **TownLevel:** NPC 상호작용 및 안전 구역.
3. **FieldLevel (핵심 개발 대상):** - A* 알고리즘 기반 몬스터 추격 AI.
   - 2D 그리드 타일 기반 이동 및 충돌 판정.
   - 특정 트리거 시 `BattleLevel`로 전환.
4. **BattleLevel:** 이미 구현된 전투 로직 실행 (필드 데이터 연동).

## 4. 코딩 컨벤션 및 제약 사항
- **Allman Style 준수:** 중괄호(`{`, `}`)는 항상 다음 줄에 배치.
- **메모리 관리:** 콘솔 환경의 자원 효율성 및 메모리 누수 방지.
- **객체 지향:** 기존 `Engine`의 `Actor-Component` 구조를 계승하여 구현.

## 5. 단계별 마일스톤
- **1일차:** `FieldLevel` 및 `TownLevel` 클래스 설계, `TileMap` 확장 (Cost/Trigger 속성).
- **2일차:** A* 알고리즘 및 BFS 이동 범위 가시화 로직 구현.
- **3일차:** 필드 내 몬스터 AI(추격) 및 플레이어 충돌 시 전투 씬 전환 브릿지.
- **4일차:** 전체 게임 루프(마을-필드-전투) 통합 테스트 및 코드 리팩토링.
